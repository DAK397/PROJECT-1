Top Command

The top command generates an ordered list of active processes. The list updates periodically and can be customized extensively to suit your requirements.

It provides vital information about:

  • Memory and CPU utilization
  • Processes and users consuming resources
  • Load averages and more

Options

Option Description
-a Sorts processes by allocated memory.
-b Starts top in batch mode for output redirection.
-c Reverses the last remembered state of command line programs.
-d Displays delay between screen updates.
-h Shows usage and library version, then quits.
-H Reverses the remembered thread state.
-i Excludes idle processes from the list.
-m Reports memory usage instead of virtual memory.
-M Displays memory in floating point units.
-n Limits the number of iterations before quitting.
-p Monitors processes by specific PIDs.
-s Runs securely without certain interactive commands.
-S Displays cumulative CPU time.
-U Filters processes by a specific user.

Sections in the Top Output

When you run the top command, the output is divided into sections:

Header Information:

  • uptime: How long the system has been running.
  • tasks: Total processes and their states (e.g., running, sleeping).
  • CPU(s): Percentage of CPU usage by system, user, and idle time.
  • Mem/Swap: Memory and swap usage.

Process Table:

The process table displays running processes with columns for:

  • PID: Process ID.
  • USER: User running the process.
  • PR: Process priority.
  • %CPU: Percentage of CPU usage.
  • %MEM: Percentage of memory usage.
  • TIME+: Total CPU time used by the process.
  • COMMAND: Name of the command or process.

Kill Command

The kill command allows you to terminate processes using their process IDs (PIDs). It is commonly used to stop unresponsive or unnecessary programs.

Signals can be sent using kill to specify the type of termination.

Options

Option Description
-l Lists all available signals.
-s Specifies the signal to send.
-9 Sends the SIGKILL signal, forcibly terminating the process.
-15 Sends the default SIGTERM signal to terminate gracefully.

Examples

  • kill -9 1234 - Forcefully kills the process with PID 1234.
  • kill -15 5678 - Gracefully terminates the process with PID 5678.

Ifconfig Command

The ifconfig command is used for configuring network interfaces and diagnosing network issues. It is commonly employed to enable or disable interfaces, assign IP addresses, and display network configurations.

Options

Option Description
up Activates the specified network interface.
down Deactivates the specified network interface.
inet Assigns an IPv4 address to an interface.
netmask Sets the netmask for the interface.
broadcast Sets the broadcast address for the interface.

Examples

  • ifconfig eth0 up - Enables the eth0 interface.
  • ifconfig eth0 192.168.1.2 netmask 255.255.255.0 - Assigns an IP and netmask to the eth0 interface.

Ping Command

The ping command tests the connectivity between two hosts by sending ICMP echo requests and receiving replies. It is commonly used to troubleshoot network issues.

Options

Option Description
-c Specifies the number of ping requests to send.
-i Sets the interval between ping requests.
-s Defines the packet size to send.

Examples

  • ping -c 4 google.com - Sends 4 ping requests to Google.
  • ping -i 2 192.168.1.1 - Sends pings to the specified IP every 2 seconds.

SSH Command

The ssh command allows secure remote login and command execution over an encrypted connection. It is widely used to manage servers and remote systems.

Options

Option Description
-p Specifies the port to connect to.
-i Specifies the identity file (private key).
-C Enables compression to speed up the connection.
-X Enables X11 forwarding for graphical applications.

Examples

  • ssh user@remote-host - Logs into the remote host as user.
  • ssh -p 2222 user@remote-host - Connects to the remote host using port 2222.
  • ssh -i ~/.ssh/id_rsa user@remote-host - Uses a specific private key for authentication.

SCP Command

The scp command is used to securely copy files between local and remote systems over an SSH connection.

Options

Option Description
-P Specifies the port for the SSH connection.
-r Recursively copies directories.
-i Specifies the identity file for authentication.

Examples

  • scp file.txt user@remote-host:/path/to/destination - Copies file.txt to the remote host.
  • scp -r folder user@remote-host:/path/to/destination - Recursively copies folder to the remote host.
  • scp -P 2222 file.txt user@remote-host:/path - Uses port 2222 to copy file.txt to the remote host.

cURL Command

The curl command transfers data from or to a server using various protocols such as HTTP, HTTPS, FTP, and more.

Options

Option Description
-o Writes output to a file instead of the terminal.
-O Uses the file name from the URL for output.
-I Fetches only the HTTP headers.
-X Specifies the HTTP method (e.g., GET, POST).

Examples

  • curl http://example.com - Fetches the content of the URL.
  • curl -o output.html http://example.com - Saves the content to output.html.
  • curl -I http://example.com - Retrieves only the headers of the URL.

Diff Command

The diff command compares two files line by line, highlighting their differences. It is useful for debugging and version control.

Options

Option Description
-u Outputs differences in unified format.
-c Outputs differences in context format.
-r Recursively compares directories.

Examples

  • diff file1.txt file2.txt - Compares the two files.
  • diff -u file1.txt file2.txt - Compares files with unified output.
  • diff -r dir1 dir2 - Recursively compares two directories.

Head Command

The head command displays the first few lines of a file. By default, it shows the first 10 lines.

Options

Option Description
-n Specifies the number of lines to display.

Examples

  • head file.txt - Displays the first 10 lines of file.txt.
  • head -n 20 file.txt - Displays the first 20 lines of file.txt.

Tail Command

The tail command prints the last few lines of a file. By default, it shows the last 10 lines.

Options

Option Description
-n Specifies the number of lines to display.
-f Follows the file as it grows (useful for logs).

Examples

  • # Display the last 10 lines of a file
    • tail file.txt
  • # Continuously monitor a log file
    • tail -f logfile.txt
